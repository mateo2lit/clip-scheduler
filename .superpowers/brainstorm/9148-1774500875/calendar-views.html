Which calendar views should we build?

Metricool offers both — which do you want for Clip Dash?

A

Month view only — improved

Rebuild the existing monthly calendar with richer post cards (thumbnails, platform icons, status badges, time). Simpler to build, great at-a-glance overview.

B

Week view only — time-based grid

7-column layout with hour rows on the left (like Metricool's detailed view). Each post card sits at its scheduled time. Best for day-by-day scheduling precision.

C

Both views — toggle between them

Month view for the big picture, week view for time precision. Toggle in the header. This is what Metricool, Buffer, and Later all do. Most complete experience.

Context — what exists today

There's already a basic month view at /calendar — it shows posts as small pills with platform color dots. We'd either improve it or replace it with something richer.
